## Основные модели обмена сообщениями
#p2p #queue #publish #subscribe #topic #pub #sub #publisher #subscriber #listener
#### Point-to-Point (P2P) — очередь (Queue)
- В этой модели сообщения отправляются в **очередь**.
- Каждое сообщение из очереди **получает только один потребитель**.
- Сообщения хранятся в очереди до тех пор, пока какой-либо потребитель их не заберёт.
- Подходит для **задач распределения работы**, когда нужно, чтобы одно сообщение обрабатывалось только один раз.
- Каждый консьюмер получает уникальные сообщения.
- Сообщения не теряются, если консьюмер временно не активен (если очередь настроена на хранение сообщений).
#### Publish/Subscribe (Pub/Sub) — топик (Topic)
- В этой модели сообщения отправляются в **топик**.
- Все подписчики (Subscribers) топика получают **копию каждого сообщения**.
- Используется для **широковещательных уведомлений**.
- Сообщения **доставляются всем подписчикам**.
- Если подписчик не активен в момент публикации, он может пропустить сообщение (если не включено durable subscription).
