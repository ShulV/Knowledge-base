#system #table #list

-  `pg_database`  информация о доступных базах данных.
-  `pg_constraint` ограничения-проверки, ограничения-исключения, а также ограничения первичного ключа, уникальности и внешних ключей, определённые для таблиц. (Ограничения столбцов описываются как и все остальные. Любое ограничение столбца равнозначно некоторому ограничению таблицы.) Ограничения на NULL представляются не здесь, а в каталоге `pg_attribute`.
-  `pg_extension` информация об установленных расширениях
- `pg_index` часть информации об индексах. Остальная информация в основном находится в `pg_class`

## system views
- `pg_stats` открывает доступ к информации, хранящейся в каталоге [`pg_statistic`](https://postgrespro.ru/docs/postgresql/9.6/catalog-pg-statistic.html "50.46. pg_statistic"). Это представление даёт доступ только к тем строкам каталога `pg_statistic`, что соответствуют таблицам, которые пользователь может читать; таким образом, это представление можно без опасений разрешить читать всем.
- `pg_user` открывает доступ к информации о пользователях базы данных. Это просто доступное для всех отображение представления [`pg_shadow`](https://postgrespro.ru/docs/postgresql/9.6/view-pg-shadow.html "50.76. pg_shadow"), в котором очищено поле пароля.
- `pg_views` даёт доступ к полезной информации обо всех представлениях в базе данных
- `pg_tables`  даёт доступ к полезной информации обо всех таблицах в базе данных.
- `pg_roles` открывает доступ к информации о ролях в базах данных. Это просто доступное для всех отображение каталога [`pg_authid`](https://postgrespro.ru/docs/postgresql/9.6/catalog-pg-authid.html "50.8. pg_authid"), в котором очищено поле пароля.
